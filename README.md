# OCRリーダー Android アプリ（実証実験版）

Androidスマートフォンのカメラで文字を読み取り、認識結果をその場で確認できるOCR実験用アプリです。  
このリポジトリは `barcodereader` をベースに、バーコード読み取りをOCR読み取りへ置き換えて作り直したものです。

---

## 何ができるか

- カメラ映像からOCRを実行
- 1回目と2回目の認識結果を照合して一致判定
- 読み取り文字数や先頭文字列の簡易チェック
- OK / NG / 重複を色と音で通知
- OCR結果をCSVログとして保存・共有

---

## 想定ユースケース

- 紙ラベルや印字メモの読み取り確認
- 帳票や伝票の文字列チェック
- 現場での簡易OCR照合
- バーコードリーダーなしでの運用検証

## 技術スタック

| 項目 | 採用技術 |
|------|---------|
| 言語 | Kotlin |
| UI | Jetpack Compose |
| カメラ | CameraX |
| OCR | ML Kit Text Recognition（Japanese） |
| ビルド | Gradle Kotlin DSL |
| パッケージ名 | `com.garyohosu.ocrreader` |

---

## 画面の流れ

```
アプリ起動
  ↓
スタート画面で読み取り回数・文字数・先頭文字列を設定
  ↓
1回目のOCR画面で文字を読み取る
  ↓
確認画面で認識結果を確認
  ↓
2回目のOCR画面で同じ対象を再読み取り
  ↓
判定画面で OK / NG / 重複 を表示
```

---

## OCRチェック設定

| 設定 | 動作 |
|------|------|
| OCR文字数（0 = 任意） | 読み取った文字数が設定値と異なる場合はエラー |
| 先頭文字列（空 = チェックなし） | 認識結果が指定文字列で始まらない場合はエラー |

## ビルド方法

### 前提条件

- Android Studio（最新安定版）
- JDK 17 以上
- Android SDK（API 23 以上）

### APK の作成

```bash
cd android-app
./gradlew assembleDebug
```

生成物:

```text
android-app/app/build/outputs/apk/debug/app-debug.apk
```

### テスト

```bash
./gradlew testDebugUnitTest
```

## 既知の制限事項

- OCR精度は照明・フォント・撮影距離に大きく左右されます
- 手書き文字や低解像度の文字は誤認識が増えます
- 画面は縦向き固定です
