# バーコード照合 Android アプリ（実証実験版）

Androidスマートフォンのカメラで2つのバーコードを連続読み取りし、一致・不一致を即時表示する実証実験用アプリです。

---

| バージョン | ダウンロード |
|-----------|------------|
| v0.3.0 | [app-debug.apk](https://github.com/garyohosu/barcodereader/releases/tag/v0.3.0) |
| v0.1.0 | [barcodereader-v0.1.0-debug.apk](https://github.com/garyohosu/barcodereader/releases/tag/v0.1.0) |

---

## 目的

スマートフォン内蔵カメラによるバーコード照合が現場作業に耐えられるかを確認します。  
出荷確認・ラベル照合・コンテナ確認などへの応用を想定した実証実験版です。

## ビルド方法

### 前提条件

- Android Studio（最新安定版）
- JDK 17 以上
- Android SDK（API 23 以上）

### APK の作成

リポジトリルートから実行する場合:

```bash
cd android-app
./gradlew assembleDebug
```

生成物:

```
android-app/app/build/outputs/apk/debug/app-debug.apk
```

### ビルド確認

```bash
./gradlew testDebugUnitTest
```

## インストール方法

### USB 経由（adb）

`android-app` ディレクトリにいる場合:

```bash
adb install app/build/outputs/apk/debug/app-debug.apk
```

リポジトリルートにいる場合:

```bash
adb install android-app/app/build/outputs/apk/debug/app-debug.apk
```

### ファイル転送

1. `app-debug.apk` をスマートフォンへ転送する（メール・クラウドストレージ等）
2. Android の「設定 → セキュリティ」で「提供元不明のアプリ」を許可する
3. ファイルマネージャーから APK をタップしてインストールする

## 操作方法

### 基本フロー

```
アプリ起動
  ↓
スタート画面 →「設定」ボタンで読み込み数・バーコード長・ヘッダーを設定
  ↓
「スタート」ボタンを押す（読み込み数が未設定の場合は無効）
  ↓
1つ目読み取り画面 → バーコードをカメラにかざす（ピッ）
  ↓
確認画面 → 読み取った値を表示。「次へ」ボタンを押して進む
  ↓
2つ目読み取り画面 → バーコードをカメラにかざす（ピッ）
  ↓
判定画面
  ├─ 一致・未登録 → 青背景「OK」＋ OK音 → 進捗 x/N 件
  ├─ 不一致      → 赤背景「NG」＋ NG音
  └─ 一致・重複  → 橙背景「重複」＋ NG音
  ↓
OK →「次のバーコードを読む」/ NG・重複 →「もう一度」
```

### バリデーション

バーコード読み取り時に以下を検証します（いずれも設定ダイアログで設定）。

| 設定 | 動作 |
|------|------|
| バーコード長（0 = 任意） | 読んだ長さが設定値と異なる場合「バーコード長が違います（読んだ長さ = x）」を表示 |
| ヘッダー（空 = チェックなし） | 先頭文字列が一致しない場合「ヘッダーが一致しません」を表示 |

### ボタン一覧

| ボタン | 画面 | 動作 |
|--------|------|------|
| スタート | スタート画面 | カメラ起動・1つ目読み取り開始（読み込み数未設定・完了時は無効） |
| 設定 | スタート画面 | 読み込み数・バーコード長・ヘッダーを設定するダイアログを開く |
| 次へ | 確認画面 | 1つ目の値を確認後、2つ目の読み取りへ進む |
| 中止 | 読み取り・確認画面 | 読み取りを中止してスタート画面へ戻る |
| 次のバーコードを読む | 判定画面（OK） | 状態をリセットして1つ目読み取りへ戻る |
| もう一度 | 判定画面（NG・重複） | 状態をリセットして1つ目読み取りへ戻る |
| スタート画面へ | 判定画面（完了時） | スタート画面へ戻る |
| CSVをダウンロード | スタート画面 | ログCSVを共有シートで送信する |
| ログをクリア | スタート画面 | ログと重複チェックセットを削除する（確認ダイアログあり） |
| システムバック | 読み取り・確認・判定画面 | 中止と同じ動作（アプリは終了しない） |

### 音のフィードバック

| タイミング | 音 |
|-----------|-----|
| 読み取り成功（1つ目・2つ目） | ピッ |
| 一致判定（OK） | ピンポン |
| 不一致判定（NG） | ブブー |

## カメラ権限について

初回起動時にカメラ権限の許可ダイアログが表示されます。

- **許可**: そのままスキャン開始
- **拒否**: スタート画面にエラーメッセージを表示。再度「スタート」を押すと再要求します
- **「二度と表示しない」を選択した場合**: Android の設定画面からこのアプリのカメラ権限を手動で許可してください

---

## 既知の制限事項

- カメラを連続使用するため、長時間使用時は端末の発熱やバッテリー消費が増える可能性があります
- 対応バーコード形式は ML Kit 依存です（EAN・QR・CODE 系など主要形式は動作確認済み）
- 画面は縦向き固定です

## 実証実験で確認したいこと

- スマートフォンのカメラでバーコードを安定して読み取れるか
- 読み取り速度が現場作業に耐えられるか
- 1つ目・2つ目の読み取り操作が分かりやすいか
- OK/NG 表示が直感的に分かるか
- 音によるフィードバックが有効か
- Bluetooth バーコードリーダーなしで運用できる可能性があるか
