# OCR読取Androidアプリ 実証実験仕様書

## 1. 目的

Androidスマートフォンのカメラを使って、2つのOCRを連続で読み取り、内容が一致しているかを判定する実証実験用アプリを作成する。

本アプリは、将来的な出荷確認・ラベル照合・コンテナ確認などへの応用を想定し、まずはスマホカメラによるOCR読み取りが実用に耐えるかを確認することを目的とする。

## 2. 開発方針

- Androidネイティブアプリとして作成する。
- APKファイルを作成し、手動でAndroidスマートフォンへインストールできるようにする。
- Google Playへの公開は行わない。
- 実証実験用のため、ログイン機能、サーバー保存、履歴検索機能は今回の対象外とする。
- まずは「読む」「比較する」「結果を分かりやすく表示する」ことを最優先とする。

## 3. 想定環境

- Androidスマートフォン
- スマートフォン内蔵カメラ
- APKによる手動インストール
- オフラインでも動作可能とする

## 4. 使用技術の候補

- Kotlin または Java
- Android Studio
- CameraX
- ML Kit Ocr Scanning

ただし、本実証実験版では §20「実装方針の追加指定」を優先し、Kotlin / Jetpack Compose / CameraX / ML Kit Ocr Scanning / Gradle Kotlin DSL で実装する。

## 5. 対象OCR

実証実験段階では、以下を読み取り対象とする。

- 一般的な1次元OCR
- QRコード
- Code128

対応可能であれば、JANコードやDataMatrixにも対応してよい。

## 6. 基本動作フロー

### 6.1 スタート画面

アプリ起動時はスタート画面を表示する。

表示内容:

- アプリ名
- 「スタート」ボタン

ユーザーが「スタート」ボタンを押すと、カメラ画面に移動する。

### 6.2 1つ目のOCR読み取り

スタートボタン押下後、カメラを起動し、以下のメッセージを表示する。

> 1つ目のOCRを読んでください

OCRを読み取ったら、以下を行う。

- 読み取り成功音を鳴らす
- 読み取ったOCR文字列を内部に保存する
- 画面メッセージを2つ目読み取り用に切り替える

### 6.3 2つ目のOCR読み取り

1つ目の読み取り後、カメラは起動したままにする。

画面には以下のメッセージを表示する。

> 2つ目のOCRを読んでください

OCRを読み取ったら、以下を行う。

- 読み取り成功音を鳴らす
- 読み取ったOCR文字列を内部に保存する
- 1つ目と2つ目の文字列を比較する
- 判定画面へ移動する

## 7. 判定仕様

### 7.1 一致判定

1つ目のOCR文字列と2つ目のOCR文字列が完全一致した場合、OKと判定する。

判定条件:

```text
ocr1 == ocr2
```

### 7.2 不一致判定

1つ目と2つ目のOCR文字列が異なる場合、NGと判定する。

判定条件:

```text
ocr1 != ocr2
```

### 7.3 空文字対策

読み取った値が空文字、null、または取得失敗の場合は、判定せず再読み取りを促す。

## 8. 判定画面

判定画面では、以下を表示する。

- 1つ目のOCR文字列
- 2つ目のOCR文字列
- 判定結果

### 8.1 OKの場合

一致している場合は、以下の表示を行う。

- 背景、または判定表示エリアを青色にする
- 大きく「OK」と表示する
- OK判定音を鳴らす

表示例:

```text
OK

1つ目: 1234567890
2つ目: 1234567890

OCRは一致しています
```

### 8.2 NGの場合

不一致の場合は、以下の表示を行う。

- 背景、または判定表示エリアを赤色にする
- 大きく「NG」と表示する
- NG判定音を鳴らす

表示例:

```text
NG

1つ目: 1234567890
2つ目: 9876543210

OCRが一致していません
```

## 9. 音の仕様

以下の3種類の音を用意する。

### 9.1 読み取り成功音

OCRを1つ読み取るたびに鳴らす。

用途:

- 1つ目読み取り成功時
- 2つ目読み取り成功時

音のイメージ:

- ピッ

### 9.2 OK判定音

2つのOCRが一致したときに鳴らす。

音のイメージ:

- ピンポン

### 9.3 NG判定音

2つのOCRが不一致だったときに鳴らす。

音のイメージ:

- ブブー

実装段階では、Android標準音、短い効果音ファイル、またはToneGeneratorなど、実装しやすい方法を選択してよい。

## 10. 操作ボタン

判定画面には以下のボタンを表示する。

### 10.1 もう一度

「もう一度」ボタンを押すと、読み取り状態を初期化し、1つ目の読み取り画面に戻る。

内部でクリアする値:

- 1つ目のOCR
- 2つ目のOCR
- 判定結果

### 10.2 戻る

可能であれば、読み取り途中でスタート画面に戻れるボタンを用意する。

## 11. 誤読み取り防止仕様

同じOCRを連続で誤検出しないように、1回読み取った直後は短時間だけ次の読み取りを無効化する。

目安:

- 読み取り後 0.5秒〜1秒 は次の読み取りを無効化する

これにより、1つ目のOCRを読み取った直後に、同じOCRを2つ目として誤読することを防ぐ。

## 12. カメラ制御仕様

カメラは、スタートボタン押下後に起動する。

1つ目を読み取った後も、カメラは停止せず、2つ目の読み取り状態に切り替える。

理由:

- カメラの再起動待ちを避けるため
- フォーカス調整の待ち時間を減らすため
- 現場での操作を速くするため

## 14. 実証実験で確認したいこと

このアプリで確認したい項目は以下とする。

- スマートフォンのカメラでOCRを安定して読み取れるか
- 読み取り速度が現場作業に耐えられるか
- 1つ目、2つ目の読み取り操作が分かりやすいか
- OK/NGの判定表示が直感的に分かるか
- 音によるフィードバックが有効か
- ラベルの角度、照明、汚れで読み取り精度に問題が出ないか
- BluetoothOCRリーダーを購入せずに運用できる可能性があるか

## 16. 最低限の完成条件

実証実験版として、以下を満たせば完成とする。

- AndroidスマートフォンにAPKでインストールできる
- スタートボタンを押すとカメラが起動する

## 20. 実装方針の追加指定

実装は以下で固定する。

- Kotlin
- Jetpack Compose
- CameraX
- ML Kit Ocr Scanning
- Gradle Kotlin DSL

初期対応OCR形式は以下に絞る。

- QR_CODE
- CODE_39
- CODE_128

音は効果音ファイルを使わず、Android標準の ToneGenerator を使用する。

- 読取成功音: TONE_PROP_BEEP
- OK音: TONE_PROP_ACK
- NG音: TONE_PROP_NACK

カメラ権限がない場合は、権限要求を表示する。
権限が拒否された場合は、スタート画面にエラーメッセージを表示する。

APKはまず debug APK でよい。
以下のコマンドでAPKを作成できること。

```bash
./gradlew assembleDebug
```

生成物は以下に作成されること。

```text
app/build/outputs/apk/debug/app-debug.apk
```

## 21. 実装時の注意

1つ目のOCRを読み取った直後に、同じOCRを2つ目として誤検出しないようにする。

対策として、読み取り成功後は1秒間スキャン結果を無視する。

ただし、同じOCRを意図的に2回読むケースもあり得るため、同じ値そのものは禁止しない。
